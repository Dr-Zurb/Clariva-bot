## üí° Learning Tips

### 1. Learn Just-In-Time
Don't try to learn everything before building. Learn what you need for the current task.

### 2. Read Code, Not Just Tutorials
The Clariva codebase is your best textbook. See how concepts apply in real code.

### 3. Build, Break, Fix
Try changing code. See what breaks. Fix it. You learn more from debugging.

### 4. Use AI as a Teacher
Ask me "why does this code work?" or "explain this pattern" when you encounter something new.

### 5. Review Your Own PRs
Before asking for code, read your changes. Check against COMMON_MISTAKES.md.
