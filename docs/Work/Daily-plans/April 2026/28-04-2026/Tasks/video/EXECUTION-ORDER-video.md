## TL;DR — read before you touch any task

1. **DO NOT execute by ID** (A1 → A2 → A3 → …). The IDs describe *grouping inside a sub-batch*, not sequence.
2. **Voice Sub-batch 0 ships first.** Four small voice tasks (~6h total) wire the patient-side video page to the companion-chat surface. Until they ship, every video task that emits a Plan 06 system message (A1, A2, B3, B5, B10, E2, …) silently degrades.
3. Within each video sub-batch, the order below respects: **hard intra-batch deps → cheap items first → schema/migration work last**.
4. If a task you're about to start says `**Depends on:** ... (HARD — ...)` and the prerequisite isn't `Status: Shipped (YYYY-MM-DD)`, **stop and ship the prerequisite first.**
5. After each task ships, update its file's `Status` line: `Drafted` → `Implementation complete (YYYY-MM-DD)` → `Shipped (YYYY-MM-DD)`. Mirror the tier change in the source product plan.

## Per-task workflow (the loop)

For every single task, regardless of sub-batch:

1. **Open the task file.** Read `Status`, `Depends on`, `Acceptance criteria`, `Files expected to touch`, `Out of scope`.
2. **Check deps.** If any `Depends on` says HARD and the prerequisite isn't `Status: Shipped`, STOP and ship that first.
3. **Set status.** `task-master set-status --id=<id> --status=in-progress` (if you're using Taskmaster) or just edit the `Status:` line in the file → `In progress (YYYY-MM-DD)`.
4. **Implement.** Touch only files listed in `Files expected to touch`; if you find yourself touching files outside that list, the task scope was wrong — pause and either expand the task file or split it.
5. **Verify acceptance.** Run all checkboxes under "Acceptance criteria" and "Manual smoke" sections. Don't ship if `mode='readonly'` checkbox isn't ticked (Plan 07 invariant).
6. **Cross-batch sanity.** If the task's `Depends on` line mentions voice or text, run a 30-sec smoke on those modalities to confirm no regression.
7. **Status close.** `Status: Implementation complete (YYYY-MM-DD)` after PR merges; flip to `Shipped (YYYY-MM-DD)` after staging smoke.
8. **Update parent docs.** Tick the row in `plan-video-consult-selected-features.md` tier-section. Update the source product plan's `[SELECTED 2026-04-29]` marker → `[SHIPPED YYYY-MM-DD]` for that item.
